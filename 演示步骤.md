## 见面时演示流程

### 第一步：展示已有报告（2 分钟）

打开之前跑好的报告，给斑马看：
- "这是昨天跑的苹果分析，7 个 AI 角色协作出的报告"
- 展示关键部分：情绪分析（Reddit/StockTwits 数据）、多空辩论、最终决策

### 第二步：现场跑一只他关心的股票（5-10 分钟）

问斑马："你最近关注哪只股票？我现场跑一个给你看。"

```powershell
# 改 test_run.py 里的股票代码，然后：
python test_run.py
```

跑的过程中给他讲解：
- "现在是基本面分析师在跑，抓财报数据"
- "这个是情绪分析师，在扫 Reddit 和 StockTwits"
- "接下来多空两队会辩论"
- "最后风控经理做压力测试，给出最终决策"

### 第三步：讲你能做的增值（3 分钟）

"这个框架本身是开源的，谁都能跑。但我能帮你做的是：

1. **每天自动跑** — 你关注的 5-10 只股票，每天早上自动分析，结果推到你微信
2. **个性化调参** — 比如你更看重基本面不看技术面，可以调权重
3. **中文报告** — 已经配好了，直接输出中文
4. **历史决策对比** — 系统自带记忆，会回头看之前建议对不对，越用越准"

## 技术细节备忘（斑马如果问）

| 问题 | 回答 |
|------|------|
| 用什么模型？ | 目前配的 LongCat-2.0-Preview，也可以换 Claude/GPT-5.4/DeepSeek |
| 数据从哪来？ | yfinance（免费），股价+财报+技术指标 |
| 情绪数据？ | Reddit + StockTwits，自动爬取 |
| 支持 A 股吗？ | 目前不支持，但可以做——换数据源就行 |
| 跑一次多久？ | 深度分析 5-10 分钟，看模型速度 |
| 跑一次多少钱？ | 取决于模型，LongCat 大概几毛到一块 |
| 准确率？ | MIT 论文里回测跑赢了 buy-and-hold，但不是保证 |

## 注意事项

- **不要说"这个很简单"** — 对他来说不简单，你的价值在于能让它跑起来+定制
- **不要主动聊产品化** — 先让他用起来，体验到价值
- **带着笔记本去** — 需要现场演示，不是 PPT 讲故事
- **记得问他的投资风格** — 他偏价值投资还是短线？关注哪些行业？这决定后续怎么定制
